Step 1 · Setup

Drop the design system into your repo

1

Unzip and copy the design system

Unzip the Lead.AI Design System download. Copy the whole folder into your repo as design-system/ at the root.

your-project/ ├── design-system/ ← paste the whole zip here │ ├── README.md │ ├── SKILL.md │ ├── colors_and_type.css │ ├── fonts/ ← Plus Jakarta Sans self-hosted │ ├── assets/ ← logo-real.png, media/ │ └── ui_kits/platform/ ← reference components ├── src/ │ └── components/ └── ...
2

Wire the font + CSS tokens into your app

In your global stylesheet (e.g. app/globals.css or src/index.css):

/* At the top of globals.css */
@import '../design-system/colors_and_type.css';

/* Then your app styles, using the vars: */
body {
  background: var(--bg-1);
  color: var(--fg-1);
  font-family: var(--font-sans);
}

If you use Tailwind, extend your config to expose the CSS vars as utilities — or just use them inline/in CSS modules.

3

Copy the wireframe into your repo too

Save wireframe.html as design-system/wireframe.html. Claude Code will read it as structural reference.

4

Verify Claude Code can see the files

Open Claude Code in your project root and run a quick test:

Test prompt
Read design-system/README.md and summarize the Lead.AI design system in 3 bullet points. Then read design-system/wireframe.html and list the 16 screens from the Table of Contents.

If it reads both correctly, you're ready.

Step 2 · Orient Claude Code

Send this as the first prompt — every session

Paste this at the start of each Claude Code session for the revamp. It loads the full context once, so you don't have to re-explain.

Master context prompt — paste once per session
You are revamping the CAPA AI pharma QA app to use the Lead.AI design system.

Read these files to orient yourself:
1. design-system/README.md — brand personality, visual foundations, content rules
2. design-system/colors_and_type.css — all design tokens (CSS vars for color, type, spacing, radii, shadows, motion)
3. design-system/wireframe.html — full UX wireframes for all 16 screens we're building
4. design-system/ui_kits/platform/kit.css — utility classes (glass, glow-field, reveal, etc.)
5. design-system/ui_kits/platform/Marketing.jsx and Console.jsx — reference component patterns

Design system summary (do not deviate from these):
- Background: near-black layered surfaces (--bg-0 to --bg-4)
- Text: near-white primary (--fg-1), muted gray secondary (--fg-2)
- Accent: SINGLE brand gradient purple→magenta→coral (#9B3AC5 → #C4688C → #E07D60)
  — Primary buttons use var(--grad-brand) background
  — Active nav items use var(--accent-soft) bg + var(--accent) text
- Typography: Plus Jakarta Sans (--font-sans) for all UI, IBM Plex Mono (--font-mono) for eyebrows/metrics/code
- Heading tracking: negative letter-spacing (-0.02em to -0.035em), semibold (600)
- Cards: --bg-2 surface, 1px --line-2 border, --r-lg radius, optional inset top-light
- Glass cards: backdrop-filter blur(22px) + rgba(255,255,255,0.07) — for content over imagery
- Motion: opacity + translateY, 500–900ms, cubic-bezier(0.22,1,0.36,1). No bounce.
- Copy: sentence case everywhere. No emoji. No exclamation marks. Calm, declarative.
- Logo: use design-system/assets/logo-real.png

App context:
- CAPA AI = pharma QA compliance app for Fill-Finish/Warehouse/QA departments
- Core user: QA Analyst (daily 8D workflow) + QA Manager (monitoring, sign-off)
- Core flow: My Work → CAPA Hub → 8D step screens (D1–D7)
- Nova = the AI assistant embedded inline in 8D steps as a suggestion block
- The wireframe defines exact layouts — implement them faithfully, styled with this DS

When I say "build screen X", implement it as a React component (or whatever the project stack uses) following the wireframe layout exactly, styled with the design system tokens. Do not invent new layouts; do not add sections not in the wireframe.

Tip: Save this in a file called design-system/CLAUDE_CONTEXT.md and just say "Read design-system/CLAUDE_CONTEXT.md to orient yourself" each session.

Step 4 · Iteration

Useful follow-up prompts

Visual polish pass

After each screen is built
Do a visual polish pass on [ScreenName]:
1. Check all text colors — primary labels should be --fg-1, secondary/supporting should be --fg-2, captions/mono labels should be --fg-3
2. Check all card borders — should be 1px var(--line-2), hover state 1px var(--line-3)
3. Check all button states — primary uses var(--grad-brand), hover adds brightness(1.1) filter, active scale(0.99)
4. Check heading letter-spacing — h1/h2/h3 should have negative tracking (-0.02em to -0.03em)
5. Verify no hardcoded hex colors — everything should use CSS vars from colors_and_type.css
6. Verify the eyebrow/kicker labels use --font-mono, uppercase, letter-spacing 0.18em, --fg-3 color

Accessibility + empty states

Prompt
Add empty states and loading skeletons to [ScreenName]:
- Empty state: --bg-2 card, centered, Lucide icon in --fg-4 (40px), h3 in --fg-1, body text in --fg-2, optional CTA button. No emoji, no illustrations.
- Loading skeleton: shimmer animation from --bg-3 to --bg-4, matching the shape of actual content cards.
- Error state: --danger-soft bg, --danger left border, error message in --fg-2.
Follow Lead.AI content voice: calm, declarative. E.g. empty CAPA list: "No CAPAs match your filters. Adjust the filters above or create a CAPA from a finding."

Motion + transitions

Prompt
Add motion to [ScreenName] following the Lead.AI motion DNA:
- Scroll/mount reveals: opacity 0→1 + translateY(12px→0), duration 600ms, easing cubic-bezier(0.22,1,0.36,1)
- Stagger sibling cards: 80ms delay between each
- Slide-over panel: translateX(100%→0), 400ms, same easing
- Hover transitions: 180ms (--dur-fast). No bounce. No spring physics.
- Tab/step transitions: opacity fade, 300ms
Do NOT add any bouncy, springy, or attention-seeking animations. Motion is slow, confident, and minimal.

Dark theme check

Prompt
Audit [ScreenName] for dark theme correctness:
- No pure white (#fff or white) backgrounds anywhere in the content area
- No pure black text (use --fg-1 which is #F3F5F7)
- All border colors should have < 20% opacity or use --line-* vars
- Inputs: --bg-4 background, never white
- Modals/popovers: --bg-3, shadow var(--shadow-lg)
- Charts: --bg-2 background, grid lines use --line-1
Fix any violations.
